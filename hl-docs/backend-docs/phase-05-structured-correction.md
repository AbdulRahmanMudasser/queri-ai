# Phase 05: Structured JSON Output & Self-Correction

## 1. Phase Purpose
This document controls execution for Phase 05, focusing on enforcing structured JSON output from Gemini, pre-executing column validation in the AST, and implementing a single-attempt query self-correction loop.

## 2. Feature Objective
* **Phase Goal:** Force Gemini to return query results in structured JSON schema, validate table/column nodes against the cached schema reader catalog, and trigger a retry correction block when query executions fail.
* **User Or System Outcome:** Eliminates raw markdown syntax errors and column hallucinations, improving overall query translation reliability.
* **In Scope:** Structured schema response bindings, `validator.py` column validation methods, and `query.py` execution retry loops.
* **Out Of Scope:** Vector embeddings and conversation history.

## 7. UX / Behavioral Expectations
* **User-Facing Change:** Failed translation queries are automatically corrected under-the-hood without showing an error to the user if the retry is successful.
* **System Behavior Change:** FastAPI endpoints catch `DBAPIError` or column catalog exceptions, forward them back to Gemini for one correction attempt, and retry.

## 8. Acceptance Criteria
- Gemini translation utilizes structured outputs mode or Pydantic validation parsing.
- Query AST validator throws `ValueError` if a column/table is not in cached schema.
- Retries are triggered on database execution failures and capped at a maximum of 1 attempt.
